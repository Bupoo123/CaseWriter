# CaseWriter - 病例报告撰写助手

## 项目简介

CaseWriter 是一个基于 CARE 2013 标准的纯前端病例报告撰写助手，旨在帮助团队成员规范化撰写病例报告（Case Report）。本工具提供结构化写作流程、写作指导和主流期刊推荐功能。

**版本**: V1.2  
**编制单位**: 杰毅生物研发中心

## 功能特性

### 核心功能

1. **撰写流程导航** - 基于 CARE 2013 指南的13个结构化章节
2. **写作指导模块** - 提供章节写作要点、范例展示和常见问题提示
3. **期刊选择与模板** - 提供20+主流 Case Report 期刊信息，支持筛选和快速跳转
4. **本地数据存储** - 使用浏览器 LocalStorage 自动保存，确保数据隐私
5. **多格式导出** - 支持导出为 Word (.doc) 和 Markdown (.md) 格式

### CARE 2013 章节结构

1. 标题
2. 关键词
3. 摘要
4. 引言
5. 患者信息
6. 临床发现
7. 时间线
8. 诊断评估
9. 治疗干预
10. 随访和结果
11. 讨论
12. 患者观点（可选）
13. 知情同意

## 技术架构

- **类型**: 纯前端网页应用（HTML + CSS + JavaScript）
- **框架**: Vue.js 3（CDN 版本）
- **数据存储**: 浏览器 LocalStorage
- **导出格式**: Word (.doc), Markdown (.md)
- **部署方式**: 静态页面（可直接部署到 GitHub Pages 或内部知识平台）

## 使用方法

### 本地运行

1. 克隆或下载项目到本地
2. 直接在浏览器中打开 `index.html` 文件
3. 无需安装任何依赖，所有资源通过 CDN 加载

### 在线部署

可以将项目部署到以下平台：

- **GitHub Pages**: 将项目推送到 GitHub 仓库，启用 Pages 功能
- **内部服务器**: 将项目文件上传到 Web 服务器即可
- **其他静态托管服务**: Vercel、Netlify 等

## 使用指南

### 1. 开始撰写

1. 打开应用后，默认进入"撰写流程"视图
2. 从左侧导航栏选择要撰写的章节
3. 在右侧编辑区输入内容
4. 系统会自动保存到浏览器本地存储

### 2. 使用写作指导

1. 点击顶部导航栏的"写作指导"
2. 查看 CARE 指南核心原则、常见错误、写作技巧和示例
3. 参考指导内容完善你的报告

### 3. 选择期刊

1. 点击顶部导航栏的"期刊选择"
2. 使用搜索框或筛选器查找合适的期刊
3. 点击"投稿链接"或"模板链接"访问期刊网站
4. 根据期刊要求调整报告格式

### 4. 保存和导出

- **自动保存**: 编辑内容时会自动保存到浏览器本地存储
- **手动保存**: 点击"保存到本地"按钮手动保存
- **加载数据**: 点击"从本地加载"恢复之前保存的内容
- **导出 Word**: 点击"导出 Word"生成 .doc 格式文档
- **导出 Markdown**: 点击"导出 Markdown"生成 .md 格式文档

## 数据隐私

- 所有数据仅存储在浏览器本地（LocalStorage）
- 不会上传到任何服务器
- 数据完全由用户控制
- 清除浏览器数据会删除保存的内容

4. **期刊信息**: 期刊信息（APC、投稿链接等）可能会变化，请以期刊官网为准
